# LaunchDarkly — Toggle Travel

This branch is instrumented with the **LaunchDarkly Observability SDK** — an SDK-based approach that requires code changes to the application. This is in contrast to the Dynatrace branch where the OneAgent instruments with zero code changes.

## What Was Changed vs `main`

- `public/js/nav.js` — injects the LaunchDarkly badge into the nav bar
- SDK integration code (see setup below)

## SDK Installation

```bash
npm install launchdarkly-node-server-sdk
```

Initialize in `src/server.js` (or a dedicated `src/ld.js` module):

```javascript
const LaunchDarkly = require('launchdarkly-node-server-sdk');
const ldClient = LaunchDarkly.init(process.env.LD_SDK_KEY);

module.exports = ldClient;
```

Add to `.env`:
```
LD_SDK_KEY=sdk-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

## What to Demo

**Feature Flags**
- Wrap the Vacation Mode feature in a flag: `ldClient.variation('vacation-mode-enabled', context, false)`
- Demo killing the AI feature instantly without a deploy — flip the flag off in the LaunchDarkly UI, reload the page, the toggle disappears
- Show how percentage rollouts work: serve Vacation Mode to only 20% of users, watch the traffic split in the dashboard

**Observability**
- Every flag evaluation is an event — show the evaluation stream in LaunchDarkly
- Use the seed script personas as named user contexts so you can see per-user flag states
- Correlate feature flag changes with the error rate events generated by the error spike flow

**The "Ship vs Kill" Demo**
- Enable a flag that increases the simulated payment failure rate (`SIMULATE_PAYMENT_FAILURE_RATE`)
- Watch error rate climb in logs
- Kill the flag → error rate drops immediately
- No code change, no deploy — shows the value of feature flags for incident response

## Key Concepts to Highlight

| Concept | Demo action |

| Instant kill switch | Turn off `vacation-mode-enabled` flag |
| Percentage rollout | Set Vacation Mode to 10% → 50% → 100% |
